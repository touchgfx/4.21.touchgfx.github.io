Getting Started

설치 섹션에 설명되어 있듯이, TouchGFX 사용을 시작하려면 먼저 최신 버전의 TouchGFX를 설치 해야 합니다.

설치가 완료되면 TouchGFX의 첫 번째 프로젝트를 시작할 준비가 됐습니다. 본 설명서의 주요 섹션들을 꼭 읽어보시고 기본적인 TouchGFX 개념을 숙지하시기 바랍니다. 가장 기본적인 섹션은 개발 소개(Development Introduction)로, 완벽한 TouchGFX 프로젝트에 필요한 소프트웨어 및 하드웨어 구성 요소를 비롯해 이들을 생성하기 위한 활동 및 도구들이 소개되어 있습니다. 이 외에도 사전 제작된 구성 요소를 사용하는 빠른 프로토타이핑 방법도 나와 있습니다.

TouchGFX Quick Start​

If you are eager to start experimenting with TouchGFX, trying out an example UI project and maybe running it on an STM32 Evaluation Kit, a wide range of TouchGFX Board Setups together with GUI Examples and Demos exists in TouchGFX Designer.

TouchGFX 보드 설정 + 예시 및 데모​

To create a TouchGFX project based on a TouchGFX Board Setup go ahead and start the TouchGFX Designer.

TBS(TouchGFX Board Setup) 및 예시를 통해 애플리케이션 생성

TouchGFX Designer의 시작 창의 다양한 UI 예시 중 원하는 것을 선택하여 시작하십시오. 이를 STM32 평가 키트의 여러가지 보드별 코드를 통합하여 임베디드 하드웨어에서 예시를 즉시 실행해 볼 수 있습니다.

STM32 평가 키트가 없는 경우에는 PC에서 간단하게 TouchGFX 애플리케이션을 실행할 수 있습니다. 첫 번째 단계에 대한 가이드는 튜토리얼 챕터에서 확인할 수 있습니다. 여기에서는 프로젝트를 설치 및 실행하여 복잡한 UI 애플리케이션을 구현하는 방법을 소개합니다.

Board Specific Demo​

TouchGFX Designer에는 TouchGFX Board Setup에 예시와 데모를 하나씩 결합하는 방식 외로, 특정 STM32 평가 키트를 위한 특별 데모들이 포함되어 있습니다. These are located in the Demos section under Board Specific Demo.

보드별 데모를 이용해 애플리케이션 생성

다음 단계​

이제 설명서 전체를 살펴보면서 자신에게 필요한 챕터들을 확인하시기 바랍니다. 이렇게 전체 설명서의 개요를 확인하여 자세한 정보의 위치를 알 수 있으며, TouchGFX 주요 개념들을 숙지할 수 있습니다.

주요 챕터들은 다음과 같습니다:

  • Basic Concepts: 이 챕터는 설명서의 나머지 부분들에 대한 배경 지식을 제공합니다. 여기서는 차후에 레퍼런스가 되는 주요 그래픽 개념들을 모두 소개합니다. 현재 갖추고 있는 지식과 앞으로 수행하게 될 TouchGFX 개발에 따라 몇몇 섹션들은 건너 뛰고 나중에 세부적인 내용이 궁금할 때 참조할 수 있습니다.
  • Development: 본 설명서의 메인 챕터입니다. TouchGFX 프로젝트 구조를 비롯해 전체 프로젝트 개발 주기에 포함되어 있는 워크플로와 도구들이 설명되어 있습니다. 워크플로의 각 단계는 자체적인 섹션으로 분류되어 자세한 설명을 확인할 수 있습니다. 여기서는 TouchGFX 프레임워크의 모든 기능과 구성 요소에 대한 설명을 포함해 하드웨어에서 애플리케이션을 실행하는 방법과 UI 개발 방법을 이해하는 데 필요한 모든 정보를 찾을 수 있습니다.
